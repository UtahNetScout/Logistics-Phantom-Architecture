# SYNCON Sample Demo Package

This folder contains a lightweight sample evidence package generated by the SYNCON runner.

**Command used:**

```bash
python syncon.py run --scenario demo --run-id syncon-demo --output-dir examples --phantoms 250 --contaminated 3 --workers 1
```

## Sample Result

| Metric | Value |
|--------|-------|
| Protected synthetic convoys | 1 |
| Phantom records generated | 250 |
| Phantom multiplier | 250x |
| Intentionally seeded unsafe phantoms | 3 |
| Approved records | 247 |
| Rejected records | 3 |
| Red-team SNR | 0.0000 |

## Included Artifacts

- `REPORT.md`
- `scenario.json`
- `validation.json`
- `red_team.json`
- `timeline.json`
- `SUMMARY.md`

`phantoms.json` is intentionally omitted from this committed example because full phantom telemetry payloads can be large. Running SYNCON locally will regenerate the complete artifact set, including `phantoms.json`.

## Scope

All data in this folder is synthetic prototype data. It is not operational telemetry, not a real convoy route, and not deployment-ready deception tooling.
